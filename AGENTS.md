# AGENTS.md — Designstoun 2026

## Описание проекта

Система учёта производства каменных изделий (резка камня). Управление партиями сырья, приёмкой готовой продукции, работниками и синхронизацией с МойСклад.

**Стек:** PHP 8.4, Laravel 12, PostgreSQL, Bootstrap 5.3, Alpine.js

---

## Как работать с проектом

### Порядок действий

1. **Перед правкой** — изучи существующий код в похожих файлах
2. **Правь код** — следуй стилю проекта, не добавляй комментарии без запроса
3. **Проверь** — запусти тесты и линтер
4. **Готово** — сообщи пользователю что сделано

### Проверки после изменений

```bash
make test      # Тесты
make lint      # Линтер
```

## Конвенции

### Архитектура

Проект использует двухслойную архитектуру:
- Controllers: обработка HTTP, вызов сервисов, формирование ответа
- Services: бизнес-логика, оркестрация операций
- Model (Eloquent): доступ к данным, запросы к БД 
- Model содержат только отношения, scopes, accessor'ы
- Все операции с МойСклад — через сервисы в `app/Services/`

Контроллеры работают только через сервисы.
Сервисы для работы с БД используют Model (Eloquent).
Прямой доступ к БД из контроллеров запрещен.

### Middleware доступа

- `WorkerOnly` — для рабочих (пильщики, галтовщики)
- `MasterOnly` — для мастеров
- Новый маршрут? Добавь в `ALLOWED_ROUTES` нужного middleware

### МойСклад API

- Имена документов через `DocumentNaming::weeklyName()`
- При `duplicate_name` — `DocumentNaming::nextSuffix()` + retry
- Ошибки возвращать как `['success' => false, 'code' => '...', 'message' => '...']`
- Текст ошибки читать из `errors[0]['error']`

### UI

- Формы: `border-radius: .4rem` для всех input/select
- Двойной рендеринг: десктоп (`d-none d-md-block`) + мобильный (`d-md-none`)
- Цвет по типу камня: `Product::getColorBySku()`
- Фильтры скрыты по умолчанию, состояние в localStorage
